### 1. Scope

#### 1.1 Identification
This document establishes the Software User Manual (SUM) for the **Rootcastle Home Watcher Stack**, Version 1.2.0.

#### 1.2 System Overview
The Rootcastle Home Watcher is an intelligent, local-first surveillance and alert appliance. It utilizes OpenCV and multimodal AI models to confirm security alerts, broadcasting snapshots via a secure Telegram interface and allowing PTZ control.

---

### 2. Operational Environment

The user controls the appliance using two main channels:
1. **Rootcastle Web Dashboard**: Accessed via Caddy proxy HTTPS on port `10443` or direct UI on port `8971`.
2. **Interactive Telegram Bot**: Accessible from any mobile device or workstation authenticated with the configured Telegram Bot Token.

### 3. Installation & Getting Started

#### 3.1 Step 1: Clone and Stage Files
Clone the repository folder to your target device (e.g. Raspberry Pi 5):
```bash
git clone https://github.com/rootcastleco/home-watcher.git
cd home-watcher
```

#### 3.2 Step 2: Run the Automated Stack Installer
Run the installation script with root permissions to install Docker, setup directories, and configure certificates:
```bash
# Standard installation
sudo bash install_frigate_raspberry_pi.sh

# Optional: Installation with isolated ESP32 access point configuration
sudo SETUP_ESP32_AP=true ESP_AP_PASSWORD='your-secure-ap-password' bash install_frigate_raspberry_pi.sh
```

#### 3.3 Step 3: Configure Local Environments
Open `/opt/frigate/.env` using a terminal text editor and fill in your network and integration parameters:
```bash
sudo nano /opt/frigate/.env
```
Key fields to modify:
- **`FRIGATE_DVR_IP`**: Local IP of the DVR system (e.g., `10.0.10.10`).
- **`FRIGATE_PTZ_IP`**: Local IP of the ONVIF PTZ camera (e.g., `10.0.10.20`).
- **`TELEGRAM_BOT_TOKEN`**: Private token generated by Telegram's `@BotFather`.
- **`TELEGRAM_CHAT_ID`**: Numeric chat ID where notifications will be sent.
- **`OPENROUTER_API_KEY`**: API key from OpenRouter.

#### 3.4 Step 4: Run Services
Start the appliance containers using Docker Compose:
```bash
cd /opt/frigate
sudo docker compose up -d
```

### 4. System Operation

#### 4.1 Accessing the Web UI
Open your web browser and navigate to the appliance IP:
- **Encrypted Ingress**: `https://<YOUR_PI_IP>:10443` (Recommended, reverse-proxied with Caddy).
- **Direct LAN Access**: `https://<YOUR_PI_IP>:8971` (Direct Frigate dashboard).

#### 4.2 Interactive Bot Commands
Send these commands directly to the authenticated Telegram bot to query or control the appliance:

| Command | Usage | Description |
|:---|:---|:---|
| `/status` | `/status` | Returns uptime, CPU/memory state, and queue details |
| `/cameras` | `/cameras` | Lists all cameras, health states, and mute times |
| `/snapshot` | `/snapshot <camera_name>` | Generates a fresh, high-resolution snapshot |
| `/pause` | `/pause <camera_name> <minutes>` | Mutes notifications for a specified camera |
| `/resume` | `/resume <camera_name>` | Instantly restores camera notifications |
| `/ptz` | `/ptz` | Generates the interactive arrow controller pad |

#### 4.3 Using the PTZ Controller Pad
1. Send `/ptz` in the chat to open the arrow pad overlay (▲, ▼, ◀️, ▶️, 🛑, and Zoom `+`/`-`).
2. Press any direction to start movement. The bot publishes a start signal and automatically issues a **`STOP`** signal after **`0.5 seconds`** to prevent camera drift.
3. The bot automatically updates the chat with a fresh snapshot from the new camera position.

### 5. Diagnostics and Troubleshooting

#### 5.1 Check Stack Logs
Check execution traces for the custom alert service or NVR daemon:
```bash
cd /opt/frigate
# Follow logs for the AI alert service
sudo docker compose logs -f security-ai-alerts

# Follow logs for the main NVR engine
sudo docker compose logs -f frigate
```

#### 5.2 Validate MQTT Broker Status
Ensure the Mosquitto broker is accepting connections:
```bash
sudo docker compose ps mqtt
```

#### 5.3 Clear Alerts Muting
If you are not receiving alerts, verify if cameras are muted. Run `/resume <camera_name>` or restart the worker:
```bash
sudo docker compose restart security-ai-alerts
```
